# Dacheng Skills 整体规划方案

> 基于 AI 能力清单 + 任务中心设计方案 + DB_SCHEMA，规划 Claude Code 可直接落地的 AI 辅助能力。
>
> **原则**：不建任务中心前端，AI 通过 Claude Code + dacheng skills 直接操作 Parse 数据，操作员在对话中下达指令。

---

## 一、能力分层

将 27 个 AI 功能点按**实现方式**分为三层：

| 层级 | 数量 | 说明 | 特点 |
|------|:----:|------|------|
| **L1 - 直接可用** | 15 | Claude Code 对话中直接执行，读写 Parse 数据 | 无需开发，只需 skill 指导 |
| **L2 - 需辅助工具** | 8 | 需要联网检索、文件解析等外部能力 | 需 MCP 工具或脚本辅助 |
| **L3 - 需前端配合** | 4 | 需要前端 UI 交互配合 | 暂缓，待任务中心建成 |

---

## 二、L1 直接可用能力（15 项）— 立即落地

这些能力 Claude Code 现在就能做，只需要对应的 skill 指导数据操作。

### 2.1 审阅与修订（4 项） — skill: `review-feedback`

| # | 能力 | 读取 | 写入 | 关键字段 |
|---|------|------|------|---------|
| 5.1 | **AI 自动审阅** | 内容表 contentHtml | `work_feedback` | sectionId, projectId, paragraphIndex, feedbackText, source=`ai`, status=`pending` |
| 5.2 | **AI 自动修订** | work_feedback + 原文段落 | `work_revision` | sectionId, projectId, paragraphIndex, revisedHtml, rationale, feedbackId, source=`ai`, status=`pending` |
| 5.3 | **修订请求处理** | work_revision（source=`ai_request`, status=`requested`）+ 关联 feedbackId + 原文 | `work_revision` | 生成新 revision，source=`ai`, status=`pending` |
| 5.4 | **版本差异总结** | `work_snapshot` × 2（同 snapshotType 不同 version） | 无（输出报告） | 对比 contentHtml / feedbacksJson / revisionsJson |

> **内容表路由**：论文正文 → `work_thesis_manuscript`，论文大纲 → `work_thesis_outline`，NSF 申请书 → `work_nsf_proposal`，NSF 科学问题 → `work_nsf_sq`

**典型对话**：
```
用户：帮我审阅张三论文的 CH1 绪论
AI：查询 work_thesis_manuscript（projectId, module='CH1', isLatest=true）→ 得到 sectionId
    → 逐段分析 contentHtml → 批量写入 work_feedback → 汇报：生成 8 条反馈意见
```

| # | 能力 | 读取 | 写入 | 关键字段 |

| 4.2 | **全章初稿生成** | 模块定义 + 文献库 | 内容表 | 旧版本 isLatest=false → 新版本 projectId, module, contentHtml, version+1, isLatest=true |
| 4.3 | **学术语言润色** | 原文 contentHtml | 内容表 | 润色后 contentHtml，版本+1 |

> **模块 key 参考**：NSF 申请书 M1-M4，NSF 科学问题 SQ0/SQ1-3/SQH/SQS，论文正文 CH1-CH5+（摘要/致谢等），论文大纲 OL_CH1-OL_CH5

**典型对话**：
```

AI：查询 work_thesis_outline（module='OL_CH1', isLatest=true）读取大纲
    → 查询 work_literature（projectId）读取文献库
    → 生成 CH1 初稿 → 写入 work_thesis_manuscript（新版本）
```

## 三、L2 需辅助工具能力（8 项）— 短期可落地

需要搭配 MCP 工具、联网搜索或文件解析。

### 3.1 文献管理（3 项） — skill: `literature-ops`（新增）

| # | 能力 | 依赖工具 | 操作 | 关键字段 |
|---|------|---------|------|---------|
| 2.1 | **AI 在线检索** | WebSearch MCP | 搜索学术文献 → 写入 `work_literature` | projectId, libraryId, title, authors, doi, year, journal, type, source=`ai_search`, entryStatus=`needs_verify` |
| 2.3 | **文献信息核验** | WebSearch / DOI API | 通过 DOI 联网查询 → 比对题录信息 | verifyMarks（v_pass/v_error/v_incomplete/v_suspected/v_manual）, entryStatus（verified/unverifiable） |
| 2.4 | **文献摘要标注** | WebFetch（文献全文） | 读取文献 → 生成摘要 + 提取关键词 | abstract, keywords, tags |

### 3.2 科学问题与大纲（2 项） — skill: `content-writing`

| # | 能力 | 依赖工具 | 操作 | 关键字段 |
|---|------|---------|------|---------|
| 3.1 | **科学问题生成** | 需跨表综合（档案 + 文献） | 分析 work_profile + work_literature → 生成 SQ0/SQ1-3/SQH/SQS | 写入 `work_nsf_sq`（每个 module 一条记录） |
| 3.3 | **论文大纲生成** | 需跨表综合（选题 + 文献） | 分析 OL_TITLE + work_literature → 生成 OL_CH1-5 | 写入 `work_thesis_outline`（每个 module 一条记录） |

### 3.3 写作增强（1 项） — skill: `content-writing`

| # | 能力 | 依赖工具 | 操作 | 关键字段 |
|---|------|---------|------|---------|
| 4.5 | **引文自动插入** | 文献库 + citation 表 | 分析内容 → 匹配文献 → 更新 contentHtml + 创建引用记录 | `work_lit_citation`（projectId, literatureId, contentType, order） |

### 3.4 档案提取（1 项） — skill: `profile-ops`（新增）

### 3.5 选题分析（1 项） — skill: `content-writing`

## 六、操作员工作流示例

### 场景 A：审阅修订闭环

```
1. 操作员：帮我审阅张三论文的 CH1 绪论
2. Claude：
   - 调用 dacheng:review-feedback
   - 读取 .env 连接 Parse
   - 查询 work_thesis_manuscript（projectId, module='CH1', isLatest=true）→ 得到 sectionId
   - 逐段分析 contentHtml，生成反馈
   - 批量写入 work_feedback（sectionId, projectId, paragraphIndex, feedbackText, source='ai', status='pending'）
   - 记录操作日志（operation-log）
   - 汇报：生成 8 条反馈意见，已写入系统

3. 操作员：第 3、5 段的意见不错，帮我生成修改稿
4. Claude：
   - 读取第 3、5 段的 feedback（paragraphIndex=2, 4）
   - 基于 feedback + 原文段落生成修改建议
   - 写入 work_revision（sectionId, projectId, paragraphIndex, revisedHtml, rationale, feedbackId, source='ai', status='pending'）
   - 记录日志
```

### 场景 B：文献检索与核验

```
1. 操作员：搜索"肠道菌群与代谢综合征"相关文献，找 10 篇近 3 年的
2. Claude：
   - 调用 dacheng:literature-ops
   - 确认目标 projectId 和 libraryId（文献库）
   - 联网检索学术文献（WebSearch MCP）
   - 写入 work_literature × 10 条（projectId, libraryId, title, authors, doi, year, journal, type, source='ai_search', entryStatus='needs_verify'）
   - 记录日志 → 汇报结果列表

3. 操作员：核验前 5 篇的 DOI 信息
4. Claude：
   - 逐条通过 DOI 联网查询 → 比对题录信息
   - 更新 verifyMarks（v_pass / v_error / v_incomplete）+ entryStatus（verified / unverifiable）
   - 记录日志
```

### 场景 C：初稿生成

```
1. 操作员：基于李四的大纲，生成 CH2 文献综述初稿，引用工作台里的文献
2. Claude：
   - 调用 dacheng:content-writing
   - 查询 work_thesis_outline（projectId, module='OL_CH2', isLatest=true）读取大纲
   - 查询 work_literature（projectId）读取文献库
   - 生成 CH2 初稿（HTML 格式，含引用标记）
   - 更新旧版本 work_thesis_manuscript（module='CH2'）isLatest=false
   - 创建新版本 work_thesis_manuscript（projectId, module='CH2', contentHtml, version+1, isLatest=true）
   - 同步创建 work_lit_citation（projectId, literatureId, contentType='manuscript', order）
   - 记录日志
```

### 场景 D：档案诊断与方向提炼

```
1. 操作员：帮我诊断王五的档案完整度
2. Claude：
   - 调用 dacheng:profile-ops
   - 查询 work_profile（profileId）→ 得到基本信息
   - 查询 work_info_publication / grant / patent / book / past_project / core_asset / preliminary_data / past_application / collaborator
   - 分析各维度完整性 → 输出诊断报告（缺失项 + 补充建议）
   - 更新 work_profile.profileCompleteness

3. 操作员：帮他提炼研究方向
4. Claude：
   - 读取 work_info_publication（论文列表）+ work_info_grant（项目经历）
   - 分析研究主题聚类 → 提炼核心方向和优势领域
   - 写入 work_profile（researchBackground, techniques）
   - 记录日志
```

### 场景 E：数据规划建议

```
1. 操作员：基于李四的论文，建议变量定义和分析方法
2. Claude：
   - 调用 dacheng:data-plan
   - 读取 work_thesis_outline（OL_TITLE + OL_CH1-5）了解研究问题
   - 建议变量定义 → 写入 work_data_plan × N 条（module='variables', varType, varName, varSource, varConstruct, varRange）
   - 推荐分析方法 → 写入 work_data_plan × N 条（module='analysis', method, purpose, chapter）
   - 推荐可视化方案 → 写入 work_data_plan × N 条（module='visualization', chartType, dataSource, description）
   - 记录日志
```
