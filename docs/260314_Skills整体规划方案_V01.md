# Dacheng Skills 整体规划方案

> 基于 AI 能力清单 + 任务中心设计方案 + DB_SCHEMA，规划 Claude Code 可直接落地的 AI 辅助能力。
>
> **原则**：不建任务中心前端，AI 通过 Claude Code + dacheng skills 直接操作 Parse 数据，操作员在对话中下达指令。

---

## 一、能力分层

将 25+ 个 AI 功能点按**实现方式**分为三层：

| 层级 | 说明 | 特点 |
|------|------|------|
| **L1 - 直接可用** | Claude Code 对话中直接执行，读写 Parse 数据 | 无需开发，只需 skill 指导 |
| **L2 - 需辅助工具** | 需要联网检索、文件解析等外部能力 | 需 MCP 工具或脚本辅助 |
| **L3 - 需前端配合** | 需要前端 UI 交互配合 | 暂缓，待任务中心建成 |

---

## 二、L1 直接可用能力（9 项）— 立即落地

这些能力 Claude Code 现在就能做，只需要对应的 skill 指导数据操作。

### 2.1 审阅与修订（3 项）

| # | 能力 | skill | 操作 |
|---|------|-------|------|
| 5.1 | **AI 自动审阅** | `review-feedback` | 读取 contentHtml → 逐段分析 → 批量写入 `work_feedback`（source='ai'） |
| 5.2 | **AI 自动修订** | `review-feedback` | 读取 feedback + 原文 → 生成修改稿 → 写入 `work_revision`（source='ai'） |
| 5.3 | **修订请求处理** | `review-feedback` | 读取 source='ai_request' 的 feedback → 生成 revision |

**典型对话**：
```
用户：帮我审阅张三论文的 CH1 绪论
AI：（读取 CH1 contentHtml → 逐段分析 → 写入 8 条 work_feedback → 汇报结果）
```

## 六、操作员工作流示例

### 场景 A：审阅修订闭环

```
1. 操作员：帮我审阅张三论文的 CH1 绪论
2. Claude：
   - 调用 dacheng:review-feedback
   - 读取 .env 连接 Parse
   - 查询 work_thesis_manuscript（projectId, module='CH1', isLatest=true）
   - 逐段分析，生成反馈
   - 批量写入 work_feedback（source='ai', status='pending'）
   - 记录操作日志（operation-log）
   - 汇报：生成 8 条反馈意见，已写入系统

3. 操作员：第 3、5 段的意见不错，帮我生成修改稿
4. Claude：
   - 读取第 3、5 段的 feedback
   - 生成修改建议
   - 写入 work_revision（source='ai', status='pending'）
   - 记录日志
```

### 场景 B：文献导入与核验

```
1. 操作员：搜索"肠道菌群与代谢综合征"相关文献，找 10 篇近 3 年的
2. Claude：
   - 调用 dacheng:literature-ops
   - 联网检索学术文献
   - 写入 work_literature × 10 条（source='ai_search', entryStatus='needs_verify'）
   - 记录日志
   - 汇报结果列表

3. 操作员：核验前 5 篇的 DOI 信息
4. Claude：
   - 逐条查询 DOI → 比对信息
   - 更新 verifyMarks + entryStatus
   - 记录日志
```

### 场景 C：初稿生成

```
1. 操作员：基于李四的大纲，生成 CH2 文献综述初稿，引用工作台里的文献
2. Claude：
   - 调用 dacheng:content-writing
   - 读取 OL_CH2 大纲内容
   - 读取 work_literature 文献库
   - 生成 CH2 初稿（HTML 格式，含引用标记）
   - 旧版本 isLatest=false，创建新版本
   - 同步创建 work_lit_citation 引用记录
   - 记录日志
```
